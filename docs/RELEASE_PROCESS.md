# Release process / Procédure de release

## English

### Version format

Use semantic-style tags when possible:

- `v1.0.0`
- `v1.0.1`
- `v1.1.0`

### Before creating a release

Check that:

- the mod was tested in game;
- the Workshop folder contains the correct PBO;
- the PBO is signed;
- the `.bisign` file matches the PBO;
- the public key is present;
- no private key is included;
- the README, LICENSE and CHANGELOG are up to date;
- the Steam Workshop page is up to date.

### Suggested release asset

For a normal release, attach:

`AutoForward-SmartSpeed_v1.0.0_Workshop.zip`

This archive should contain the public mod folder:

`@AutoForward-SmartSpeed`

### GitHub release title

`Auto-Forward Smart Speed v1.0.0`

### GitHub release notes

Use `release_notes/RELEASE_NOTES_v1.0.0.md`.

## Français

### Format de version

Utilisez des tags de type sémantique lorsque c’est possible :

- `v1.0.0`
- `v1.0.1`
- `v1.1.0`

### Avant de créer une release

Vérifiez que :

- le mod a été testé en jeu ;
- le dossier Workshop contient le bon PBO ;
- le PBO est signé ;
- le fichier `.bisign` correspond au PBO ;
- la clé publique est présente ;
- aucune clé privée n’est incluse ;
- le README, la licence et le changelog sont à jour ;
- la page Steam Workshop est à jour.

### Archive de release conseillée

Pour une release normale, ajoutez :

`AutoForward-SmartSpeed_v1.0.0_Workshop.zip`

Cette archive doit contenir le dossier public du mod :

`@AutoForward-SmartSpeed`

### Titre de release GitHub

`Auto-Forward Smart Speed v1.0.0`

### Notes de release GitHub

Utilisez `release_notes/RELEASE_NOTES_v1.0.0.md`.
